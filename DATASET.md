# Clean Dataset Schema — `master_daily.csv` & `cycles.csv`

Generated by `preprocess.py` from the raw mcPHASES CSVs.

## Summary

| Output | Rows | Columns | Purpose |
|--------|------|---------|---------|
| `master_daily.csv` | 5,659 | 54 | One row per (participant, study_interval, day_in_study). Use for daily-level models and feature sequences. |
| `cycles.csv` | 125 | 12 | One row per detected menstrual cycle. Use for cycle-length / period-duration regression. |

**Coverage**
- 42 participants, 2 study intervals (2022 & 2024).
- **125 period episodes detected** — avg ~3 per participant.
- **83 complete cycles** (start + next start observed) with a mean cycle length of **28.1 ± 8.6 days** (median 29) — consistent with clinically typical cycles (21–35 days).
- **period_duration**: mean 5.6 days, median 5 — consistent with normal menstruation length (3–7 days).

## `master_daily.csv` — column reference

### Keys
| Column | Description |
|--------|-------------|
| `id` | Participant identifier |
| `study_interval` | 2022 or 2024 |
| `day_in_study` | Day index within the interval (1 = first day) |
| `is_weekend` | Boolean |

### Derived labels (targets for ML)
| Column | Description |
|--------|-------------|
| `is_bleeding` | 1 if `flow_volume` was anything above "Not at all" |
| `period_start` | 1 on the first day of a new period (bleeding day preceded by ≥ 7 quiet days) |
| `cycle_id` | Running index of cycles per (id, interval); 0 before the first detected period_start |
| `day_in_cycle` | Days since the most recent `period_start`; NaN before the first one |
| `days_to_next_period` | **Primary daily regression target** — days until the next `period_start`; NaN if right-censored |

### Self-report & hormones (from `hormones_and_selfreport.csv`)
| Column | Description |
|--------|-------------|
| `phase_num` | 0 = Menstrual, 1 = Follicular, 2 = Fertility, 3 = Luteal |
| `lh` | Luteinizing hormone (mIU/mL) |
| `estrogen` | Estrone-3-glucuronide (ng/mL) |
| `pdg` | Pregnanediol glucuronide (mcg/mL) |
| `flow_volume_num` | 0–7 ordinal (0 = Not at all … 7 = Very Heavy) |
| `flow_color_num` | Same ordinal scale applied to flow color label |
| `{symptom}_num` for symptoms: `appetite`, `exerciselevel`, `headaches`, `cramps`, `sorebreasts`, `fatigue`, `sleepissue`, `moodswing`, `stress`, `foodcravings`, `indigestion`, `bloating` | Likert 0–5 (0 = Not at all … 5 = Very High) |

### Fitbit physiological signals (daily means)
| Column | Source | Description |
|--------|--------|-------------|
| `rhr` | `resting_heart_rate.csv` | Resting heart rate (bpm) |
| `active_sedentary` / `active_lightly` / `active_moderately` / `active_very` | `active_minutes.csv` | Minutes at each activity intensity |
| `fitbit_stress_score` | `stress_score.csv` | Daily stress score |
| `sleep_overall_score`, `sleep_composition_score`, `sleep_revitalization_score`, `sleep_duration_score`, `sleep_deep_sleep_in_minutes`, `sleep_resting_heart_rate`, `sleep_restlessness` | `sleep_score.csv` | Overnight sleep quality metrics |
| `resp_full_sleep_breathing_rate`, `resp_deep_sleep_breathing_rate`, `resp_rem_sleep_breathing_rate` | `respiratory_rate_summary.csv` | Breathing rates by sleep stage |
| `hrz_below_default_zone_1`, `hrz_in_default_zone_1`, `hrz_in_default_zone_2`, `hrz_in_default_zone_3` | `time_in_heart_rate_zones.csv` | Minutes in HR zones |
| `temp_nightly_temperature`, `temp_baseline_relative_nightly_standard_deviation` | `computed_temperature.csv` | Skin temperature & SD vs. baseline — valuable for ovulation/luteal detection |

Slowly-changing physiological columns (`rhr_*`, `sleep_*`, `temp_*`, `resp_*`) are forward-filled within (id, interval) with a max gap of 3 days.

### Participant baseline (static, joined from `subject-info.csv`)
| Column | Description |
|--------|-------------|
| `birth_year` | Year of birth |
| `age_of_first_menarche` | Age at first period |
| `literacy_num` | Menstrual health literacy: 0 Non-existent → 4 Expert |
| `sexually_active_num` | 1 = Yes, 0 = No |
| `age_at_interval` | `study_interval − birth_year` |

## `cycles.csv` — column reference

One row per detected cycle. Use this for cycle-length regression (primary prediction task).

| Column | Description |
|--------|-------------|
| `id`, `study_interval` | Participant & interval |
| `cycle_index` | 1-based index of the cycle within the interval for that participant |
| `start_day_in_study` | Day the cycle began |
| `next_start_day_in_study` | Day the next cycle began (NaN if right-censored) |
| `cycle_length` | **Primary target** — days between this period_start and the next one |
| `period_duration` | Length of the bleeding episode (days), merging gaps ≤ 2 days |
| `birth_year`, `age_of_first_menarche`, `literacy_num`, `sexually_active_num`, `age_at_interval` | Baseline participant covariates |

## Quickstart — fit a baseline model

```python
import pandas as pd
from sklearn.ensemble import GradientBoostingRegressor
from sklearn.model_selection import GroupKFold
from sklearn.metrics import mean_absolute_error

cycles = pd.read_csv("cycles.csv").dropna(subset=["cycle_length"])

features = ["age_at_interval", "age_of_first_menarche", "literacy_num",
            "sexually_active_num", "period_duration", "cycle_index"]
X = cycles[features].fillna(cycles[features].median())
y = cycles["cycle_length"]
groups = cycles["id"]

maes = []
for train, test in GroupKFold(n_splits=5).split(X, y, groups):
    m = GradientBoostingRegressor(random_state=0).fit(X.iloc[train], y.iloc[train])
    maes.append(mean_absolute_error(y.iloc[test], m.predict(X.iloc[test])))
print("CV MAE (days):", sum(maes) / len(maes))
```

A stronger model adds the last 7–14 days of `rhr`, `temp_nightly_temperature`, `sleep_*` aggregates leading up to `start_day_in_study` as cycle-level features (rolling mean + std).

## Predicting the full period **date range**

Combine two models to produce the requested range:

1. **Start-date model**: regress `cycle_length` on cycle-level features → `predicted_next_start = last_period_start + predicted_cycle_length`.
2. **Duration model**: regress `period_duration` on the same features → `predicted_end = predicted_next_start + predicted_duration − 1`.

Final output:
```
predicted_range = [predicted_next_start, predicted_end]
```

## Notes / caveats
- `cycle_length` is right-censored for the last cycle of every (id, interval) — drop those rows for training.
- 2022 and 2024 intervals are treated as independent time series; cycle metrics never span intervals.
- `pdg` is almost entirely missing in the raw data — safe to drop as a feature.
- Participants have 1–5 cycles each, so personalized models need pooling/hierarchical structure.
